# 测试框架说明（中文版）

本文档描述测试框架（`run_test_harness.py`）中包含的测试。`tests.h` 中还定义了更多测试，可以在 `main.cpp` 的 `main` 函数中找到。这些测试可以通过 `../part_*/runtasks` 二进制文件直接运行。

## SuperSuperLight（超超轻量）
此测试分配两个各含 2^15 个元素的缓冲区，然后将元素从一个缓冲区复制到另一个缓冲区，每次迭代反转复制顺序。共64个任务和400次批量任务启动。

## SuperLight（超轻量）
此测试分配两个各含 2^15 个元素的缓冲区，以交替顺序将元素从一个缓冲区复制到另一个。每个任务在复制数据到输出缓冲区之前会递增输入缓冲区中的值。共64个任务和400次批量任务启动。

## PingPongEqual（等量乒乓）
此测试分配两个各含 2^19 个元素的缓冲区，以交替顺序复制元素。每个任务在复制前递增输入缓冲区中的值。工作在任务之间**均匀分配**。共64个任务和400次批量任务启动。

## PingPongUnequal（不等量乒乓）
与 `pingPongEqual` 相同，但某些任务分配的工作比其他任务多。

## RecursiveFibonacci（递归斐波那契）
此测试使用计算密集型的递归方法计算斐波那契数。共30次批量任务启动，每次256个任务。

## MathOperationsInTightForLoop（紧密循环数学运算）
每个任务不接收输入，执行32次涉及指数、对数、乘法和加法运算的计算密集型计算。每次批量启动16个任务，输出数组大小为512，共2000次批量启动。对于有依赖关系的测试，每个任务依赖于前一个任务。

## MathOperationsInTightForLoopFewerTasks（紧密循环数学运算-更少任务）
与 `MathOperationsInTightForLoop` 相同，但将每次批量启动的512个工作单元仅分配给9个任务。因此，每个任务获得更大的计算份额，且某些任务的工作略多于其他任务。

## MathOperationsInTightForLoopFanIn（紧密循环数学运算-扇入）
此测试分为两类任务。第一类是计算密集型数学运算（与 `MathOperationsInTightForLoop` 相同类型）。第二类是末尾的单一加法归约操作。归约依赖于所有计算密集型数学任务。每次批量数学运算启动64个任务（输出数组2048），共256次批量启动。归约操作对所有批量启动的输出向量进行向量求和。

## MathOperationsInTightForLoopReductionTree（紧密循环数学运算-归约树）
与 `MathOperationsInTightForLoopFanIn` 类似，但采用二叉树结构进行加法归约。每个加法归约对两次批量任务启动的结果求和。每个加法归约依赖于两个前序操作。数学运算每次批量启动64个任务（输出数组16384），共32次批量启动。每个加法归约输入32768大小的数组，输出16384大小的数组。

## SpinBetweenRunCalls（run调用间自旋）
首先启动一个轻量任务的批量启动（仅复制一个值）。其次启动2个中等权重任务（计算第40个斐波那契数）。最后再启动一个轻量任务。在异步情况下，最终任务依赖于前两个任务。

## MandelbrotChunked（Mandelbrot分块）
此测试使用单次批量启动中的128个任务计算Mandelbrot分形图像，将问题分解为生成连续输出图像行块的任务。计算本身是计算密集型的。注意，由于只执行一次批量任务启动，线程池和每次run()创建线程应该具有相似的性能。
